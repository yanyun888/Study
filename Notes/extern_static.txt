
extern 和 static 关键字

- 静态全局变量和普通全局变量
静态全局变量：
	+ 不必担心其他源文件使用相同的变量名，彼此相互独立。
	+ 在某源文件中定义的静态全局变量不能被其他源文件使用或修改。
	+ 只能在本文件中使用
普通全局变量：
	+ 
	
	
	
一般地，使用static变量时，都会放在源文件中声明并定义，为了防止不必要的信息污染；使用extern变量时，在同文件中进行声明，在源文件中进行定义。

一个变量不能同时由static和extern进行修饰。

extern和const
C++ 中 const修饰的全局常量具有跟static的相同特性，即他们只能作用于本编译模块中；但是extern与const连用声明的常量可以作用与其他编译模块中。


const char* g_str = “123456” 与 const char g_str[]="123456" 并不相同
第一个表达式修饰的是char* 而不是g_str，如果希望得到第二个表达式的效果，需要
这样写 const char* const g_str = “123456”
